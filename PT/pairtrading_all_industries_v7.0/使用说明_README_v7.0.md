### 核心机制讲解
1. 北交所过滤机制
```bash
def is_bse_stock(stock_code):
    """北交所股票代码特征"""
    code_str = str(stock_code)
    return code_str.startswith(('83', '87', '88', '92'))
```
过滤逻辑：
在解析行业文件时自动识别并剔除
控制台输出：剔除北交所: XX只
2. 滑点计算机制

| 操作 | 公式                         | 示例（股价10元） |
| 买入 | 成交价 = 市价 × (1 + 0.0003) | 10.003元   |
| 卖出 | 成交价 = 市价 × (1 - 0.0003) | 9.997元    |

影响：
单次交易滑点成本：0.03%
完整交易（买+卖）：0.06%
报告输出：slippage_impact 字段显示累计影响
3. 行业筛选流程
遍历所有行业
    ↓
剔除ST股票 + 北交所股票
    ↓
样本内筛选Top 5候选配对（SSD + 协整检验）
    ↓
样本外回测所有候选配对
    ↓
【关键】只保留收益率 > 0 的配对
    ↓
【关键】取收益率前2名（不足2对则剔除该行业）
    ↓
进入行业相关性监控

4. 三层行业监控
| 层级   | 监控内容                  | 阈值       | 失败处理   |
| 第一层 | 相关性质量（稳定性+一致性） | 质量分>0.5 | 警告并检查  |
| 第二层 | 尾部风险（危机同跌概率）    | λ<0.5      | 剔除较差行业 |
| 第三层 | 风险平价权重               | 动态计算    | 自动分配   |

5. 资金分配机制
双层分配架构：

总资金：1000万
    ↓
【第一层：行业间风险平价】
    行业权重 = 1/行业风险 / Σ(1/行业风险)
    ↓
【第二层：行业内等权重】
    配对资金 = 行业资金 / 2（每行业2对）

示例：
行业	风险指标	行业权重	行业资金	每对资金
银行	0.12	     35%	   350万	   175万
医药	0.18	     23%	   230万	   115万
电子	0.25	     17%	   170万	   85万
...	...	...	...	...

## 调整滑点
```bash
# 激进估计（流动性差）
'SLIPPAGE': 0.0005  # 万分之五

# 保守估计（流动性好）
'SLIPPAGE': 0.0001  # 万分之一
```
## 调整行业筛选严格度
```bash
# 更严格（只保留高收益行业）
'MIN_POSITIVE_PAIRS': 3  # 至少3对正收益
'FINAL_TOP_N': 1          # 每行业只选1对

# 更宽松（更多行业参与）
'MIN_POSITIVE_PAIRS': 1   # 至少1对正收益
'FINAL_TOP_N': 3          # 每行业选3对
```
## 调整风险平价敏感度
修改 RiskParityAllocator.calculate_weights()：
```bash
# 原版：纯风险平价
inverse_risks = {k: 1/max(v, 0.001) for k, v in industry_risks.items()}

# 改为：风险平价+收益调整（风险收益比）
inverse_risks = {k: (returns[k]/max(v, 0.001)) for k, v in industry_risks.items()}
```
### 🔍常见问题
Q1: 为什么很多行业被剔除？
可能原因：
- 正收益配对不足（检查 MIN_POSITIVE_PAIRS）
- 尾部风险过高（行业间相关性监控触发）
- 数据质量问题（检查通达信数据完整性）
排查方法：
```bash
# 查看详细日志，观察每个行业的处理结果
python pairtrading_all_industries_v7.0.py 2>&1 | tee backtest.log
```

Q2: 滑点设置多少合理？
参考标准：
| 股票类型          | 建议滑点        | 说明    |
| 大盘股（>500亿）    | 0.01%-0.03% | 流动性好  |
| 中盘股（100-500亿） | 0.03%-0.05% | 流动性一般 |
| 小盘股（<100亿）    | 0.05%-0.10% | 流动性差  |
建议：回测用0.03%，实盘建议0.05%-0.10%
Q3: 如何只回测特定行业？
修改主函数中的循环：
```bash
# 只回测银行和医药
target_industries = ['银行', '医药']
for idx, (industry_name, stock_codes) in enumerate(industry_map.items(), 1):
    if industry_name not in target_industries:
        continue
    # ... 原有逻辑
```
Q4: 回测速度很慢怎么办？
优化建议：
1. **减少候选配对数**：TOP_N_PAIRS_PER_INDUSTRY = 3
2. **缩短回测周期**：只回测最近1年数据
3. **并行处理**:（需修改代码，使用multiprocessing）
Q5: 如何验证滑点影响？
查看输出中的 slippage_impact 字段：
slippage_impact > 1%: 滑点影响过大，建议检查流动性
slippage_impact 0.3%-1%: 正常范围
slippage_impact < 0.3%: 滑点控制良好

## 🐛 故障排除
| 现象            | 可能原因          | 解决方案                                    |
| `未找到行业文件` | 路径错误          | 检查 `LOCAL_INDUSTRY_TXT_PATH`              |
| `所有行业被剔除` | 数据缺失或日期错误 | 检查通达信数据完整性，确认日期范围             |
| `滑点成本>2%`    | 交易过于频繁      | 调整 `max_holding_days` 或 `exit_threshold` |
| `选中行业<3个`   | 市场系统性下跌    | 延长样本外周期，或放宽 `MIN_POSITIVE_PAIRS`   |
| `内存不足`       | 行业数量过多      | 分批回测，或减少 `TOP_N_PAIRS_PER_INDUSTRY`  |
